Known Bugs and Limitations

Interaction sheet display failure

In some cases the interaction sheet may not display when Fink is waiting for a response. If you experience this problem, select Tools:Interact with Fink to enter a response. An optional Interact with Fink item is also available for the Toolbar. In addition, please send me an email at sburrious@users.sourceforge.net describing the prompt for which no interaction sheet was displayed. I will add it to the list of prompts that FinkCommander looks for when running commands.

Apt-get update failure

If you receive an error message after running apt-get update, try running Source:Scanpackages and then run apt-get update again.

Atlas install problem

The atlas package install process includes a long interactive session where it's necessary to answer several questions from the command line. When FinkCommander reaches this point in the process, it stalls and doesn't even print the prompts. I'm not certain why this occurs, but using the FinkCommander interaction dialog to respond to all of the questions presented during the atlas install would in any case be tedious. If you want to install atlas, you will need to do so in Terminal, either directly or by running Source:Run in Terminal:Install in FinkCommander.

Tcsh and term-readkey-pm package problems

Selection of tcsh or term-readkey-pm triggers a warning, because FinkCommander stalls when trying to install these packages from source. Installing the binary or installing source in the Terminal should work fine. Please file a bug report if you encounter the same problem with another package.
