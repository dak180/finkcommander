Overview

FinkCommander provides a GUI frontend to the Fink package manager. You will need to install Fink itself in order for FinkCommander to be of any use.

FinkCommander is designed to be intuitive. A fairly extensive set of help files is nevertheless included.

If you encounter bugs, it would be very helpful to have them reported at the SourceForge Bug Tracker for FinkCommander.

I will not be able to provide direct support for FinkCommander. Software development is my hobby. I have a pretty demanding "day job," so responding directly to every request for help is just not in the cards. I will, however, take your comments into account in designing and drafting documentation for future versions of FinkCommander. Please send comments (other than bug reports) to my SourceForge email account.

Note on Security

FinkCommander does not use Apple's Security Framework to authorize commands requiring administrator privileges. Instead, it asks you to enter your password in a text field and then passes it along to the sudo command.

Please note that FinkCommander does not itself run as root. Instead, it uses the sudo command to run Fink as root, just as Fink itself does. I believe that the main concern about this approach is that you are giving your password to FinkCommander. If FinkCommander were evil software, it could use the password do mischief to your system.

On the other hand, FinkCommander is free software, so what it does with your password is transparent. If you have concerns, I would encourage you to obtain and look at the source code to confirm what the program is doing with your password.

I have been studying the Security Framework and Apple's recommended method for running privileged operations, as exemplified in the AuthSample code sample. It appears that using this method to perform the complex inter-process communication that FinkCommander engages in will be extremely difficult, at least for an amateur programmer like me. FinkCommander is likely to remain in its present form indefinitely.
