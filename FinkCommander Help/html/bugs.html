Known Bugs and Limitations

Terminating commands

There is currently no way to terminate a command other than to quit FinkCommander. In my experience, Fink has done an excellent job of recovering from aborted commands. YMMV, of course.

Automatic response to prompt failures

In some instances, Fink will not automatically respond to a prompt when Preferences:General:Always choose default response to prompts is selected. When this occurs, select Tools:Interact with Fink to enter a response. More detail.

Incorrect error messages

FinkCommander sometimes reports possible errors in running a fink command when none has in fact occurrred. However, when a possible error is detected, FinkCommander performs a full update of the table. If the table reflects what you would expect to see after a successful command (e.g., the package you just installed from source is shown as current), then the error message was a false positive. You don't need to worry about it.

Binary install failures

When you try to use Binary:Install to install some packages identifed in the table as binary, FinkCommander will generate an error message. If this happens, you will need to use Source:Install to install the package. More detail.

Apt-get update failure

If you receive an error message after running apt-get update, try running Source:Scanpackages and then trying apt-get update again.

Incorrect table entries

If you turn off the Alway update table with Fink option, FinkCommander will attempt to update the data in the table, but the result will be at best a very rough approximation of the true state of your installation. To get a complete update, turn the option back on, or run File:Update table. More detail.

Tcsh package problem

The selection of tcsh is disabled, because FinkCommander stalls when trying to install this package. Installing from the command-line works fine. Please file a bug report if you encounter the same problem with another package.
