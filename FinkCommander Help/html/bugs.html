Known Bugs and Limitations

Terminating Commands

There is currently no way to terminate a command other than to quit FinkCommander.

Incorrect error messages

FinkCommander sometimes reports possible errors in running a fink command when none has in fact occurrred. However, when a possible error is detected, FinkCommander performs a full update of the table. If the table reflects what you would expect to see after a successful command (e.g., the package you just installed is shown as current, then the error message was a false positive.

Binary install failures

When you try to use Binary:Install to install some packages identifed in the table as binary, FinkCommander will generate an error message. If this happens, you will need to use Source:Install to install the package. More detail.

Incorrect table entries

If you turn off the Alway update table with Fink option, FinkCommander will attempt to update the data in the table, but it will be at best a very rough approximation. To get a complete update, turn the option back on, or run File:Update table. More detail.

Tcsh package problem

The selection of tcsh is disabled, because FinkCommander stalls when trying to install this package. Installing from the command-line works fine.
