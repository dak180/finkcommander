Running Fink and Apt-Get Commands

Basics

Running commands is straightforward. If the command is package specific (e.g. "install"), select a package or packages in the table and choose the command you want from the source or binary menus or the toolbar. To select more than one package, just shift- or command-click as you would in any other table

To run a command that doesn't apply to specific packages (e.g. "selfupdate"), simply choose the command from the menu.

The toolbar has a number of available commands that are not included in the default configuration. To customize the toolbar, select Window:Customize Toolbar.

When a command is running, the name of the command and (space allowing) a list of the packages it applies to will appear as a text message in the lower left-hand corner of the window. When the command is finished, a progress bar and the message "Updating table data" may appear. More detail. While either of these processes is underway, the menu and toolbar items for running commands will be disabled. Between commands, the number of packages currently displayed by the table will appear.

Source vs. Binary

The commands under the Source menu are run by the fink package manager. The commands under the Binary menu are run by apt-get. In general, fink handles packages available as source code and apt-get handles packages available in binary form. Hence the menu names.

This dichotomy, however, does not always hold true. For example, if you install a source code package, remove it and then reinstall it before a newer version becomes available, fink will use the dpkg command to reinstall a compressed binary package that it has installed on your system.

Finding Packages

FinkCommander provides two handy tools for finding the packages you might be interested in installing or removing:

  1. You can sort the table by clicking on the column headers, just as you can in Mail and other applications. Click a header twice in a row to reverse the sort order.

  2. The toolbar in its default configuration includes a filter that can be applied to most of the fields in the table. (I didn't think anyone would want to filter by version number.) The filter works incrementally, so that as you type or delete characters, the contents of the table change to include only those packages that satisfy your search criteria. The package count at the bottom of the window also changes to reflect the table contents.

By combining the use of sorting and filtering, you can effectively search the table with two criteria.

The table colums, by the way, can be resized and moved, and any such changes you make will be saved between runtimes.
