Configuring FinkCommander and Fink in Preferences

The FinkCommander Preferences Panel is divided into two tab views: General and Fink. The General Preferences tab allows you to configure FinkCommander's behavior. The Fink Preferences tab allows you to change the fink configuration settings stored in the fink-basepath/etc/fink.conf file.

This help file describes the purpose of the preference items that may not be self-evident.

The General Preferences Tab

Never ask for password

Select this option if you have configured sudo not to ask for your password. The instructions for doing so are described in a Fink FAQ

Always update table with Fink

This option causes FinkCommander to perform a full update of the table after running any command that could affect the state of your packages. More detail.

Path to Fink files

Only change the settings here if FinkCommander was unable to locate the path to your Fink installation on startup.

The Fink Preferences Tab

Use unstable packages/cryptography packages

You can use these items to switch easily and quickly between using unstable packages and not using them.

For example, if you ordinarily want to stick with stable packages, but would like to try a particular package from unstable, select the use unstable option and click OK. The table will update itself to show the unstable packages. Install the one you want and then de-select the use unstable item to go back to your previous configuration.

Use passive mode FTP/HTTP proxy/FTP proxy

Set these items to the appropriate values if you are running Fink from behind a firewall. What are the appropriate values? Heck if I know.
