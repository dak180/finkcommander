Verwenden von Fink and Apt-Get Kommandos

Grundlagen

Kommandos ausführen ist selbsterklärend. Wenn sich ein Kommando auf ein Paket bezieht (z.B. "Paket installieren"), dann markieren Sie ein oder mehrere Paket(e) in der Tabelle und wählen aus dem Source oder Binär Menü das Kommando, welches Sie ausführen möchten. Verwenden Sie Umstelltaste- oder Optionstaste-Klick um mehr als ein Paket zu markieren

Um ein Kommando auszuführen, welches nicht paketabhängig ist (z.B. "Aktualisieren der Paketliste") wählen Sie einfach das Kommando von Menü oder Werkzeugleiste.

Die Werkzeugleiste stellt eine Anzahl Symbole zur Ausführung von Kommandos zur Verfügung, welche in der Grundkonfiguration nicht enthalten sind. Um die Werkzeugleiste Ihren Bedürfnissen anzupassen, öffnen Sie Window:Customize Toolbar. Generell sind Symbole für Operationen mit binären Paketen blau, solche für Operationen mit Paketen, welche direkt von Quelldateien erstellt werden, grün.

Wenn ein Befehl ausgeführt wird, erscheint der Name des Kommandos und, falls dies der Platz erlaubt, eine Liste der Pakete auf welche der Befehl angewendet wird, linksunten im Fenster. Sobald das Kommando beendet ist erscheint entweder die Meldung "Fertig" oder "Paketdaten werden jetzt aktualisiert". Mehr Details.

Solange sich Kommandos in Ausführung befinden, werden die entsprechenden Menüeinträge und Werkzeugleisten-Symbole ausgeblendet.

Kommandos zum Beenden von Prozessen

Sie können einen Prozess gewöhnlich beenden durch (1) Drücken des Stop-Symbols in der Werkzeugleiste, (2) Drücken von Option-. auf der Tastatur oder (3) Auswahl des Menüeintrags Werkzeuge:Befehl beenden. Beim Beenden von laufenden Fink-Prozessen von FinkCommander aus sollten Sie Vorsicht walten lassen. Mehr Details. Falls das "Beenden" Kommando nicht funktioniert müssen Sie die Unterprozesse von FinkCommander manuell beenden. Mehr Details.

Source vs. Binär

Die Kommandos im Menü Source werden durch den Fink Paket Manager ausgeführt, Kommandos im Menü Binär durch Apt-Get. Generell übernimmt Fink Pakete, welche als Quellcode heruntergeladen werden, Apt-Get Pakete, die in binärer Form verfügbar sind. Daher auch die Menünamen.

Zu jeder Regel gibt es natürlich auch Ausnahmen: Falls Sie zum Beispiel ein nur als Sourcecode erhältliches Paket installieren, es danach wieder entfernen und es erneut installieren bevor eine neuere Version erhältlich wird, so verwendet der Fink Paket Manager das Kommando dpkg, da das Paket nun ja in binärer Form, als .deb Datei, auf Ihrem System vorliegt.

Befehle im Terminal ausführen

Wenn Sie ein Befehl vom Menü Source:Im Terminal ausführen oder dem Menü Binär:Im Terminal ausführen auswählen, so öffnet sich ein neues Terminalfenster, in welchem der ausgewählte Befehl ausgeführt wird. Möglicherweise werden Sie hierbei aufgefordert Ihr Passwort einzugeben. Dieselbe Option ist auch über das Kontextmenü der Pakettabelle und einer Anzahl zusätzlicher Werkzeugleisten-Symbole erreichbar.

Falls Sie wünschen können Sie diese Option verwenden, um ein zweites fink Kommando auszuführen, während ein anderes Kommando in FinkCommander ausgeführt wird. Seien Sie bei der parallelen Ausführung von Kommandos aber vorsichtig: Gleichzeitiges Installieren eines Paketes und Entfernen einer Abhängigkeit desselben Paketes kann unvorhersehbare Folgen haben.
