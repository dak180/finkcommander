Bekannte Fehler und Einschränkungen

Ausführen von FinkCommander von einem Benutzerkonto aus, das keine Administrator Privilegien hat

Leider ist dies zurzeit nicht möglich. FinkCommander verwendet das sudo Kommando um Aktionen, die Administrator Privilegien benötigen, auszuführen, und ich habe noch keinen Weg gefunden neben dem Paswort auch den Namen eines Administrators an sudo weiterzureichen.

Fehler bei Aufforderung zur Benutzereingabe

In einigen Fällen kann es vorkommen, dass die Aufforderung zur Benutzereingabe nicht angezeigt wird wenn Fink auf eine Antwort wartet. Der Grund hierfür ist, dass Fink manchmal nicht automatisch auf eine Eingabe reagiert, obwohl die Option Einstellungen:Commander:Bei Fragen immer Vorschlag von Fink als Antwort verwenden angewählt ist. Mehr details. Falls dieses Problem auftritt, wählen Sie Werkzeuge:Eingabe an Fink senden um eine Antwort einzugeben. Optional ist hierfür auch ein Symbol für die Symbolleiste vorhanden.

Fehler bei Aktualisierung mit apt-get

Falls Sie eine Fehlermeldung erhalten, nachdem Sie apt-get update ausgeführt haben, versuchen Sie erst Source:Scanpackages auszuführen und dann erneut apt-get update.

Falsche Tabelleneinträge

Falls Sie die Option Tabelle immer mit Fink aktualisieren abgewählt haben, versucht FinkCommander selbst die Daten der Tabelle zu aktualisieren, aber das Resultat wird höchstens eine ungenaue Annäherung des wahren Zustandes sein. Um eine komplette Aktualisierung zu erhalten, müssen Sie entweder die Option wieder einschalten oder Ablage:Daten aktualisieren ausführen. Mehr Details.

Tcsh Paket Problem

Die Auswahl von tcsh ist inaktiv, da FinkCommander abstürzt wenn versucht wird, dieses Paket zu installieren. Die Installation von der Kommandozeile funktioniert hingegen problemlos. Falls Sie dieses Problem auch mit anderen Paketen haben, senden Sie bitte einen Fehlerreport.
