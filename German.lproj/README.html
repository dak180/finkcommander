Übersicht

FinkCommander ist ein graphisches Interface zum Fink Paket-Manager. Um FinkCommander zu verwenden müssen Sie zuerst Fink installieren

FinkCommander soll eine intuitive Bedienung erlauben. Dennoch enthält FinkCommander eine umfangreiche Dokumantation.

Falls Sie Fehler im Programm finden, wäre es hilfreich, wenn Sie diese durch den Bug Tracker bei SourceForge melden.

Leider bin ich nicht im Stand, direkten Support für FinkCommander anzubieten, da Software Entwicklung lediglich ein Hobby darstellt und ich daneben einen Beruf ausübe, der mich auch zeitlich fordert und es mir verunmöglicht, auf jede Anfrage sofort zu antworten. Ich versuche jedoch alle Anregungen zu berücksichtigen und sie in zukünftige Versionen von FinkCommander einfliessen zu lassen. Senden Sie Ihre Kommentare (keine Fehlermeldungen bitte) an mein SourceForge E-Mail Konto.

Bemerkung zur Sicherheit

FinkCommander verwendet nicht Apple's Sicherheits Framework zur Authentifizierung von Kommandos, welche die Rechte eines Systemadministrators verlangen. Stattdessen werden Sie aufgefordert Ihr Passwort in ein Textfeld einzugeben, von wo es direkt an das sudo Kommando weitergeleitet wird.

FinkCommander selbst läuft nicht als root sondern verwendet das sudo Kommando, wie dies auch fink selbst macht. Ich denke die grössten Bedenken bei diesem Ansatz sind, dass Sie Ihr Passwort an FinkCommander weitergeben. Wenn FinkCommander böse Software wäre, könnte das Programm Ihr Passwort schädigend für Ihr Computersystem einsetzen.

Andererseits ist FinkCommander freie Software, so dass Transparenz gewährleistet ist, was das Programm mit Ihrem Passwort macht. Falls Sie dennoch Zweifel haben ermutige ich Sie, den Quellcode zu studieren.

Ich habe das Sicherheits Framework und Apple's Vorschlag zur Authentifizierung von Kommandos, die Privilegien eines Systemadministrators fordern, anhand des Beispieles AuthSample angeschaut. Es scheint mir, dass die Anwendung dieser Methode auf die komplexe inter-prozess Kommunikation, die FinkCommander verwendet, extrem schwierig zu implementieren ist - jedenfalls für einen Amateurprogrammierer wie mich. Es ist daher wahrscheinlich, dass FinkCommander auch in Zukunft in dieser Form weiterbesteht.
